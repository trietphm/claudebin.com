# Claudebin Web App Design

**Date:** 12 December 2025
**Status:** Initial Design

## Overview

Next.js 15 app (App Router) hosted at `claudebin.link` for viewing and sharing Claude sessions.

## App Structure

```
packages/web/
├── app/
│   ├── @[username]/
│   │   └── [sessionId]/
│   │       ├── page.tsx          # Session viewer (SSR)
│   │       └── llms.txt/
│   │           └── route.ts      # Markdown endpoint
│   ├── cli/
│   │   └── auth/
│   │       └── page.tsx          # OAuth callback for CLI
│   ├── page.tsx                  # Homepage
│   └── layout.tsx                # Root layout
├── components/
├── lib/
│   └── supabase.ts               # Supabase client setup
└── public/
```

## Routes

### Homepage (`/`)

**Purpose:** Landing page with sign-in option

**Content:**
- Simple hero: "Share your Claude coding sessions"
- "Sign in with GitHub" button
- Optional: Public feed of recent sessions (can skip for MVP)

**Implementation:**
- Server component
- Minimal styling with Tailwind
- No complex features

### Session Viewer (`/@[username]/[sessionId]`)

**Purpose:** Display a published Claude session

**Functionality:**
- Fetch session from Supabase by username + session ID
- Verify access: if private, check viewer is owner (404 otherwise)
- Render conversation in minimalist reader format

**UI Features:**
- **Metadata bar:** Avatar, username, publish date
- **Progressive disclosure:** Long messages/tool outputs collapsed by default, "Show more" to expand
- **Syntax highlighting:** Code blocks highlighted with shiki or similar
- **Clean typography:** Readable fonts, good spacing, mobile-friendly
- **Chronological layout:** User messages, assistant responses, tool calls in order

**Access Control:**
- Public sessions: accessible to everyone
- Private sessions: only accessible to owner (checked server-side)
- Return 404 if session not found or access denied

**Performance:**
- Server-side render (SSR)
- Fetch from Supabase on each request (caching can be added later)
- JSONB query is fast enough for MVP

### Markdown Endpoint (`/@[username]/[sessionId]/llms.txt`)

**Purpose:** Provide markdown version of session for LLMs/tools

**Functionality:**
- Route handler (not page component)
- Fetch same session data
- Convert `conversation_data` JSONB to markdown
- Return as `text/plain` with proper headers

**Markdown Format:**
```markdown
# Session Title

Published by @username on Dec 12, 2025

---

## User
[First user message]

## Assistant
[First assistant response]

## Tool: Read
[Tool call details]

## Tool Result
[Tool output]

...
```

**Access Control:**
- Same as session viewer: public sessions accessible to all, private only to owner

### CLI Auth Callback (`/cli/auth`)

**Purpose:** Handle OAuth flow for CLI authentication

**Flow:**
1. User lands here from CLI with `?code=xyz123`
2. If not authenticated, show "Sign in with GitHub" button
3. After GitHub OAuth completes (via Supabase Auth):
   - Create profile if new user
   - Generate CLI access token
   - Show token to user: "Authenticated! Return to your terminal."
4. CLI polls Supabase or user pastes token

**Implementation:**
- Page component with Supabase Auth UI
- Handle profile creation on first sign-in
- Display success state with token

## Components

### SessionViewer
- Renders conversation from JSONB
- Handles message types: user, assistant, tool calls, tool results
- Progressive disclosure logic
- Syntax highlighting for code

### MessageBlock
- Individual message display
- Collapsible content for long messages
- Timestamp formatting

### CodeBlock
- Syntax highlighted code
- Copy button
- Language label

## Styling

- **Tailwind CSS** for utility-first styling
- **Typography Plugin** for readable text
- **Minimalist theme:** Clean, no distractions
- **Dark mode:** Optional for later, start with light

## Dependencies

- **Next.js 15** - App Router, server components
- **@supabase/ssr** - Supabase for Next.js
- **Tailwind CSS** - Styling
- **shiki** - Syntax highlighting
- **date-fns** - Date formatting

## Performance Considerations

- **SSR:** Good SEO, no client-side data fetching
- **Caching:** Can add later if needed (Supabase + Next.js cache)
- **JSONB queries:** Fast enough for MVP, no need for pre-rendering
- **Code splitting:** Next.js handles automatically

## Error States

- **Session not found:** Clean 404 page
- **Private session (unauthorized):** Same 404 (don't leak existence)
- **Network errors:** Show friendly error message
- **Malformed JSON:** Graceful degradation with error boundary
