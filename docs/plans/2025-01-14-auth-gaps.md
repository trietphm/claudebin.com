## Task 1: Add Scheduled Session Cleanup

**Problem:** `cli_auth_sessions` table accumulates expired rows.

**Files:**
- Create: `supabase/migrations/20250114100001_add_session_cleanup.sql`

**Implementation:**

```sql
-- Cleanup function
CREATE OR REPLACE FUNCTION cleanup_expired_cli_auth_sessions()
RETURNS void AS $$
BEGIN
  DELETE FROM cli_auth_sessions
  WHERE expires_at < NOW() - INTERVAL '1 hour';
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

GRANT EXECUTE ON FUNCTION cleanup_expired_cli_auth_sessions TO postgres;
```

**Schedule via Supabase Dashboard:**
1. Enable `pg_cron` extension in Database > Extensions
2. Run in SQL Editor:
```sql
SELECT cron.schedule(
  'cleanup-cli-auth-sessions',
  '0 * * * *',
  $$SELECT cleanup_expired_cli_auth_sessions()$$
);
```

**Testing:** Insert expired session, run cleanup manually, verify deletion.

## Task 2: Add Server-Side Logout

**Problem:** Plugin logout only deletes local config. Tokens remain valid.

**Files:**
- Create: `packages/web/lib/supabase/admin.ts`
- Modify: `packages/web/src/trpc/routers/auth.ts` (add `logout` procedure)
- Modify: `packages/plugin/mcp/src/tools/logout.ts` (call server first)

**Step 1: Create admin client**

```typescript
// packages/web/lib/supabase/admin.ts
import { createClient } from "@supabase/supabase-js";

export const createAdminClient = () => {
  return createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!,
    { auth: { autoRefreshToken: false, persistSession: false } }
  );
};
```

**Step 2: Add logout procedure**

```typescript
// In auth.ts router
logout: publicProcedure
  .input(z.object({ access_token: z.string() }))
  .mutation(async ({ input }) => {
    const supabase = createAdminClient();
    await supabase.auth.admin.signOut(input.access_token);
    return { success: true };
  }),
```

**Step 3: Update plugin logout tool to call server first**

**Testing:** Authenticate, logout, verify token is revoked server-side.
